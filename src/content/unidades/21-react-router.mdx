## Ruteo del lado del cliente

En las webs tradicionales, cada clic en un enlace pedía una **página nueva al servidor**. El **ruteo del lado del cliente** cambia la URL y renderiza el componente **inmediatamente**, sin recargar.

<div class="proscons">
<div class="proscons__col pro"><p class="proscons__h">Ventajas</p><ul><li>Experiencia más rápida (no se recarga todo).</li><li>Permite transiciones y animaciones.</li><li>No se vuelve a evaluar todo el HTML/CSS/JS.</li></ul></div>
<div class="proscons__col con"><p class="proscons__h">Cómo se logra</p><ul><li>Librería <code>react-router-dom</code>.</li><li>Un <code>&lt;BrowserRouter&gt;</code> que envuelve la app.</li><li>Enlaces con <code>&lt;Link&gt;</code> (no <code>&lt;a&gt;</code>).</li></ul></div>
</div>

```bash
npm install react-router-dom
```

## Las piezas básicas

<div class="deflist">
<div class="deflist__row"><div class="deflist__t">BrowserRouter</div><div class="deflist__d">Envuelve toda la app y habilita el ruteo del lado del cliente.</div></div>
<div class="deflist__row"><div class="deflist__t">Routes</div><div class="deflist__d">Contenedor de rutas. Renderiza <strong>solo la primera</strong> que coincida con la URL.</div></div>
<div class="deflist__row"><div class="deflist__t">Route</div><div class="deflist__d">Una regla: <code>path</code> (URL) + <code>element</code> (componente a mostrar).</div></div>
<div class="deflist__row"><div class="deflist__t">Link</div><div class="deflist__d">Navega sin recargar: <code>&lt;Link to="/acerca"&gt;</code>.</div></div>
</div>

```jsx
import { BrowserRouter, Routes, Route } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Inicio />} />
        <Route path="/acerca" element={<Acerca />} />
        <Route path="/detalle/:id" element={<Detalle />} />
        <Route path="*" element={<Error404 />} />
      </Routes>
    </BrowserRouter>
  );
}
```

<div class="deflist">
<div class="deflist__row"><div class="deflist__t">/detalle/:id</div><div class="deflist__d">Ruta <strong>dinámica</strong>: <code>:id</code> matchea cualquier valor (<code>/detalle/42</code>).</div></div>
<div class="deflist__row"><div class="deflist__t">path="*"</div><div class="deflist__d">Comodín: atrapa cualquier URL no encontrada → página 404.</div></div>
</div>

<Callout type="tip" title="Routes elige la PRIMERA coincidencia">
`<Routes>` recorre las rutas en orden y renderiza solo la primera que matchea la URL actual. Por eso el comodín `*` (404) va siempre **al final**.
</Callout>

<Callout type="warn" title="¿Por qué <Link> y no <a>?">
Un `<a href>` hace una **petición nueva al servidor** y **recarga toda la página**, perdiendo el estado de React (volver a montar todo). `<Link to>` intercepta el clic, cambia la URL con la History API y renderiza el componente **sin recargar** ni perder el estado. Por eso adentro de una SPA se usa siempre `<Link>`.
</Callout>

Cuando varias subrutas comparten un **layout común**, se usan **rutas anidadas** + el componente **`<Outlet />`**, que marca el “hueco” donde se renderiza la ruta hija:

```jsx
// Las rutas hijas se anidan dentro de la ruta padre
<Routes>
  <Route path="/admin" element={<Admin />}>
    <Route path="usuarios" element={<Usuarios />} />
    <Route path="configuracion" element={<Configuracion />} />
  </Route>
</Routes>
```

```jsx
// Admin.jsx — el layout común con el <Outlet /> donde entra el hijo
import { Outlet, Link } from 'react-router-dom';

export default function Admin() {
  return (
    <div>
      <h2>Panel de administración</h2>
      <nav>
        <Link to="usuarios">Usuarios</Link> | <Link to="configuracion">Configuración</Link>
      </nav>
      <hr />
      <Outlet /> {/* aquí se renderiza Usuarios o Configuracion */}
    </div>
  );
}
```

<Callout type="analogia" title="Outlet = el hueco del marco">
Pensá la ruta padre como un **portarretrato** (con su menú y título fijos) y `<Outlet />` como el **hueco** donde cambia la foto. Navegás entre hijos y solo cambia el contenido del hueco, no el marco.
</Callout>

## Los hooks de React Router

### useParams — leer parámetros de la URL

```jsx
import { useParams } from 'react-router-dom';

// Para la ruta /productos/:categoria/:id
export default function Producto() {
  const { categoria, id } = useParams();
  return <p>Categoría: {categoria} · ID: {id}</p>;
}
// En /productos/electronica/1 → categoria="electronica", id="1"
```

<Callout type="warn" title="useParams siempre devuelve strings">
`useParams()` devuelve un **objeto** cuyos valores son **siempre strings**, aunque la URL traiga un número: en `/productos/electronica/1` da `{ categoria: "electronica", id: "1" }` — `id` es `"1"` (string), no `1`. Si lo necesitás como número, convertilo con `Number(id)`.
</Callout>

### useNavigate — navegar por código

```jsx
import { useNavigate } from 'react-router-dom';

const navigate = useNavigate();

<button onClick={() => navigate('/contactos')}>Contactos</button>

// con replace: no agrega entrada al historial (no se puede "volver")
navigate('/404', { replace: true });
```

<div class="deflist">
<div class="deflist__row"><div class="deflist__t">useParams</div><div class="deflist__d">Devuelve un objeto con los parámetros dinámicos de la ruta (<code>:id</code>).</div></div>
<div class="deflist__row"><div class="deflist__t">useNavigate</div><div class="deflist__d">Devuelve una función para navegar programáticamente (tras un login, submit…).</div></div>
<div class="deflist__row"><div class="deflist__t">useLocation</div><div class="deflist__d">Da la ubicación actual (ruta, query, estado). Útil para redirección condicional.</div></div>
<div class="deflist__row"><div class="deflist__t">&lt;Navigate /&gt;</div><div class="deflist__d">Componente que redirige al renderizarse (wrapper de useNavigate).</div></div>
</div>

<Callout type="info" title="Cuál uso, y qué hace replace">
Tres preguntas deciden: ¿es un enlace que el usuario clickea? → **`<Link>`**. ¿Navego desde código tras un evento (un submit exitoso)? → **`useNavigate()`**, el enfoque **recomendado en React Router v6** (reemplaza al viejo `useHistory` de v5). ¿Quiero que el solo hecho de renderizarse redirija (usuario no logueado)? → **`<Navigate>`**. En `navigate('/404', { replace: true })`, `replace` **reemplaza** la entrada actual del historial en vez de agregar una nueva: el usuario no puede "volver" con el botón Atrás.
</Callout>

## Patrón: ruta protegida

Combinando `useLocation` + `<Navigate>` se logra el patrón clásico: si no estás logueado, te manda al login y, tras loguearte, te devuelve a donde ibas.

```jsx
import { Navigate, useLocation } from 'react-router-dom';

function RutaProtegida({ children, usuario }) {
  const location = useLocation();
  if (!usuario) {
    return <Navigate to="/login" state={{ from: location }} replace />;
  }
  return children;
}

// Uso
<Route path="/perfil" element={
  <RutaProtegida usuario={usuario}><Perfil /></RutaProtegida>
} />
```
