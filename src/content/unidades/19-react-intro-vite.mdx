## ¿Qué es React?

**React** es una **librería** de JavaScript para construir **interfaces de usuario**, creada por **Facebook en 2013**. Su ecosistema incluye **React Native**, que con los mismos conceptos permite construir apps **móviles** (iOS/Android).

<Callout type="warn" title="Librería, no framework">
La pregunta de fondo es **quién manda sobre la estructura del proyecto**: en un *framework* (como **Angular**) manda la herramienta, que impone estructura, convenciones e invoca tu código; en una *librería* mandás vos. React **no impone** nada: te da las piezas para la UI y vos decidís el resto. Por eso es una **librería**, no un framework.
</Callout>

### Sus 3 características centrales

<div class="conceptos cols-3">
<article class="concepto"><div class="concepto__t">Declarativo</div><div class="concepto__d">Describís cómo se ve cada estado; React actualiza la UI cuando los datos cambian.</div></article>
<article class="concepto"><div class="concepto__t">Basado en componentes</div><div class="concepto__d">La UI se arma con piezas independientes y reutilizables.</div></article>
<article class="concepto"><div class="concepto__t">Virtual DOM</div><div class="concepto__d">Manipula un DOM virtual en memoria y aplica solo los cambios mínimos al DOM real.</div></article>
</div>

## El problema que resuelve: JSX

Crear la UI manipulando el DOM “a mano” es **verboso** y difícil de mantener:

```js
// JavaScript puro: crear todo nodo por nodo
const titulo = document.createElement('h1');
titulo.innerText = 'Bienvenidos';
document.getElementById('root').appendChild(titulo);
```

React permite escribir **JSX**: una extensión de sintaxis que mezcla HTML con JavaScript, mucho más legible:

```jsx
const titulo = 'Bienvenidos';
const header = (
  <header>
    <h1>{titulo}</h1>
  </header>
);
```

<div class="deflist">
<div class="deflist__row"><div class="deflist__t">{'{ }'}</div><div class="deflist__d">Las llaves “escapan” de JSX hacia JavaScript: meten variables y expresiones.</div></div>
<div class="deflist__row"><div class="deflist__t">className</div><div class="deflist__d">En JSX se usa <code>className</code> en vez de <code>class</code> (es palabra reservada de JS).</div></div>
<div class="deflist__row"><div class="deflist__t">cerrar tags</div><div class="deflist__d">JSX es más estricto que HTML: hasta <code>&lt;br /&gt;</code> debe cerrarse.</div></div>
<div class="deflist__row"><div class="deflist__t">un solo raíz</div><div class="deflist__d">Debe haber un único elemento raíz (o un Fragment <code>&lt;&gt;…&lt;/&gt;</code>).</div></div>
</div>

<Callout type="tip" title="Vite hace esto por vos">
Con **Vite** no configurás Babel a mano: transpila el JSX automáticamente (vía esbuild) cada vez que guardás. Por eso podés escribir JSX libremente.
</Callout>

<Callout type="info" title="¿JSX es obligatorio?">
**No.** Como JSX es solo azúcar para `React.createElement`, podrías escribir esas llamadas a mano y React funcionaría igual — pero nadie lo hace porque JSX es mucho más legible. El **segundo argumento** de `createElement` son las **props** del elemento (`null` cuando no hay): `React.createElement('h1', null, 'Hola Mundo')`.
</Callout>

## Crear el proyecto con Vite

**Vite** es el bundler moderno que reemplazó a `create-react-app`: arranque instantáneo, recarga en caliente (HMR) y soporte nativo de ESM y JSX.

```bash
npm create vite@latest
# Framework: React · Variant: JavaScript
cd mi-app-react
npm install
npm run dev   # http://localhost:5173
```

| | create-react-app | **Vite** |
|---|---|---|
| Arranque | Lento | Instantáneo |
| Compilador | Webpack | esbuild |
| HMR | Más lento | Ultra rápido |
| Config | Compleja | Mínima |

Hay dos formas de definirlos:

<div class="proscons">
<div class="proscons__col con"><p class="proscons__h">Clases (legado)</p><ul><li>Extienden <code>React.Component</code>, usan <code>render()</code> y <code>this.props</code>.</li><li>Anticuadas desde los hooks (React 16.8).</li></ul></div>
<div class="proscons__col pro"><p class="proscons__h">Funciones (actual)</p><ul><li>Una función que devuelve JSX.</li><li>Más simples, rápidas y fáciles de testear.</li><li>Estado vía <strong>hooks</strong>.</li></ul></div>
</div>

```jsx
// Componente funcional (la forma moderna)
function Saludo(props) {
  return <h1>Hola, {props.nombre}!</h1>;
}

// Uso — los nombres de componentes van en Mayúscula
function App() {
  return (
    <div>
      <Saludo nombre="Juan" />
    </div>
  );
}
```

<Callout type="warn" title="Mayúscula obligatoria">
Los componentes **siempre** empiezan con **mayúscula** (`<Saludo />`), para distinguirlos de las etiquetas HTML que van en minúscula (`<div>`). Si lo escribís en minúscula, React cree que es una etiqueta HTML.
</Callout>

### Props y desestructuración

Las **props** son los datos que un componente padre le pasa a un hijo. Se pueden **desestructurar** para escribir más limpio:

```jsx
// Sin desestructurar
function MiComp(props) {
  return <p>{props.nombre}</p>;
}

// Desestructurando en los parámetros (forma preferida)
function MiComp({ nombre, edad }) {
  return <p>{nombre} tiene {edad} años</p>;
}
```

## Estilos en React

<div class="conceptos">
<article class="concepto"><div class="concepto__t">className + CSS</div><div class="concepto__d">Importás un <code>.css</code> y usás <code>className</code>. Soporta clases dinámicas con ternarios.</div></article>
<article class="concepto"><div class="concepto__t">Estilos en línea</div><div class="concepto__d">Un objeto JS con propiedades en <strong>camelCase</strong>: <code>backgroundColor</code>, no <code>background-color</code>.</div></article>
</div>

```jsx
import './estilos.css'; // CSS externo: import al inicio del componente

// className + clase condicional
<div className={isActive ? 'active-class' : 'inactive-class'}>…</div>

// Estilo en línea: objeto con camelCase
<div style={{ backgroundColor: 'red', fontSize: '16px' }}>…</div>
```

<div class="deflist">
<div class="deflist__row"><div class="deflist__t">llave externa</div><div class="deflist__d">Escapa de JSX hacia JavaScript (como cualquier <code>{'{…}'}</code>).</div></div>
<div class="deflist__row"><div class="deflist__t">llave interna</div><div class="deflist__d">Es un <strong>objeto literal</strong> JS con los estilos. Por eso hay <strong>dos pares</strong>: <code>{'{ }'}</code> (escape) + <code>{'{ objeto }'}</code>.</div></div>
<div class="deflist__row"><div class="deflist__t">camelCase</div><div class="deflist__d"><code>backgroundColor</code> y no <code>background-color</code>: las claves son identificadores JS, y el guion se leería como una resta.</div></div>
<div class="deflist__row"><div class="deflist__t">import CSS</div><div class="deflist__d">Un <code>.css</code> externo se trae con <code>import './estilos.css'</code> y se aplica con <code>className</code>.</div></div>
</div>
