## ¿Qué es Express?

**Express** es un framework de desarrollo web para **Node.js**. Es **minimalista** y **flexible**: te da lo justo y necesario para levantar un servidor web o una **API REST** de forma rápida, y deja el resto de las decisiones en tus manos.

<Callout type="info" title="“Unopinionated” (no dogmático)">
Express **no te impone** una arquitectura ni convenciones. A diferencia de frameworks como Angular o NestJS, vos decidís cómo organizar carpetas, capas y patrones (MVC, basado en middlewares, etc.). Más libertad, pero también más responsabilidad.
</Callout>

## Instalación

```bash
npm install express --save
```

Un servidor mínimo (“Hola Mundo”):

```js
const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => {
  res.send('¡Página de inicio!');
});

app.listen(port, () => {
  console.log(`App escuchando en el puerto ${port}`);
});
```

```js
app.get('/articulos',    (req, res) => res.send('buscando'));   // leer
app.post('/articulos',   (req, res) => res.send('agregando'));  // crear
app.put('/articulos',    (req, res) => res.send('actualizando'));// modificar
app.delete('/articulos', (req, res) => res.send('eliminando')); // borrar
```

<Callout type="tip" title="app.use() responde a TODOS los verbos">
Si en lugar de un método específico usás `app.use(ruta, handler)`, ese handler responde a **cualquier** verbo HTTP (GET, POST, PUT…). Se usa típicamente para el “catch-all” de página no encontrada.
</Callout>

<Callout type="warn" title="Desde el navegador sólo probás GET">
Escribir una URL en el navegador siempre dispara un **GET**. Para probar POST/PUT/DELETE necesitás `curl`, **Postman** o **Thunder Client**.
</Callout>

- **`req` (request):** representa la petición HTTP. Contiene toda la info que mandó el cliente.
- **`res` (response):** representa la respuesta que devolvés. Tiene métodos para enviar datos.

### Por dónde llega la info del cliente

<div class="deflist">
<div class="deflist__row"><div class="deflist__t">req.params</div><div class="deflist__d">Parámetros <strong>dentro</strong> de la ruta: <code>/usuario/:id</code> → <code>req.params.id</code></div></div>
<div class="deflist__row"><div class="deflist__t">req.query</div><div class="deflist__d">Parámetros del <strong>query string</strong> tras el <code>?</code>: <code>/art?activo=true</code> → <code>req.query.activo</code></div></div>
<div class="deflist__row"><div class="deflist__t">req.body</div><div class="deflist__d">El <strong>cuerpo</strong> de la petición (POST/PUT). Requiere <code>express.json()</code> para leerlo.</div></div>
<div class="deflist__row"><div class="deflist__t">req.headers</div><div class="deflist__d">Las <strong>cabeceras</strong> HTTP (Content-Type, Authorization, cookies…).</div></div>

## Respondiendo peticiones

En una API la mayoría de las respuestas serán **JSON** con un **código de estado** adecuado:

```js
res.send('<p>HTML o texto</p>');     // envía cualquier cosa
res.json({ usuario: 'tobi' });       // envía JSON (Content-Type correcto)
res.status(201).json({ id: 5 });     // status + JSON, encadenable
res.sendStatus(404);                 // sólo el código de estado
```

<Callout type="tip" title="status() es encadenable">
`res.status(code)` devuelve el mismo `res`, por eso podés encadenar: `res.status(200).json(...)`. Es la forma idiomática de responder con código + datos.
</Callout>

## params vs query: la confusión clásica

Antes de la tabla, mirá cómo se reparte una URL real: lo que va **en la ruta** son `params`; lo que va **después del `?`** es `query`.

| | **req.params** | **req.query** |
|---|---|---|
| ¿Dónde va? | Dentro de la ruta | Después del `?` |
| Se declara con | `:nombre` en el `path` | No se declara en el path |
| Ejemplo URL | `/articulos/15` | `/articulos?nombre=aire&activo=false` |
| Se accede | `req.params.id` | `req.query.nombre` |
| Sirve para | Identificar **un** recurso | Filtrar, ordenar, paginar |

```js
// PARAMS — un parámetro dentro de la ruta
app.get('/articulos/:id', (req, res) => {
  res.send('Se solicitó el artículo ' + req.params.id);
});

// PARAMS — múltiples parámetros
app.get('/ventas/articulo/:idArt/cliente/:idCli', (req, res) => {
  const { idArt, idCli } = req.params;
  res.send(`Ventas del artículo ${idArt} para el cliente ${idCli}`);
});

// QUERY — filtros tras el "?"
// URL: /articulos?nombre=aire&activo=false
app.get('/articulos', (req, res) => {
  res.send(`Nombre: ${req.query.nombre} · Activo: ${req.query.activo}`);
});
```

<Callout type="analogia" title="Una forma de recordarlo">
**params = la dirección de tu casa** (identifica exactamente cuál): `/casa/42`.
**query = instrucciones extra para el cartero** (cómo/qué): `/casa/42?tocar=timbre&dejar=en-portería`.
</Callout>

<Callout type="parcial">
- Express = framework **minimalista** y **no-dogmático** de Node.js para web y **APIs REST**.
- Ruta = `app.METHOD(PATH, HANDLER)`. `app.use()` atrapa **todos** los verbos.
- Cada handler recibe **`req`** (lo que llega) y **`res`** (lo que devolvés).
- La info del cliente llega por **params, query, body, headers**.
- `req.params` → dentro de la ruta (`:id`) · `req.query` → tras el `?`.
- Para responder: `res.send`, `res.json`, `res.status(code)`, `res.sendStatus(code)`.
</Callout>
