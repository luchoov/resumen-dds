## ¿Por qué React Hook Form?

Manejar formularios con `useState` campo por campo es tedioso. **React Hook Form (RHF)** simplifica y **optimiza** la creación de formularios: validación, manejo de errores y **rendimiento** (menos re-renders), todo apoyado en hooks.

<div class="conceptos">
<article class="concepto"><div class="concepto__t">Validación fácil</div><div class="concepto__d">Reglas propias o con librerías como Yup/Joi.</div></article>
<article class="concepto"><div class="concepto__t">Performance</div><div class="concepto__d">Minimiza renders innecesarios → más rápido.</div></article>
<article class="concepto"><div class="concepto__t">Manejo de errores</div><div class="concepto__d">Mensajes de error simples de mostrar y gestionar.</div></article>
<article class="concepto"><div class="concepto__t">Integrable</div><div class="concepto__d">Con Material-UI, Ant Design, Bootstrap…</div></article>
</div>

```bash
npm install react-hook-form
```

## Los componentes principales

Todo sale del hook **`useForm`**:

<div class="deflist">
<div class="deflist__row"><div class="deflist__t">useForm</div><div class="deflist__d">Hook principal. Inicializa la lógica y devuelve todo lo demás.</div></div>
<div class="deflist__row"><div class="deflist__t">register</div><div class="deflist__d">Registra un campo y le define reglas de validación. Lo conecta al formulario.</div></div>
<div class="deflist__row"><div class="deflist__t">handleSubmit</div><div class="deflist__d">Maneja el envío: valida y, si está OK, llama a tu función con los datos.</div></div>
<div class="deflist__row"><div class="deflist__t">formState</div><div class="deflist__d">Estado del formulario: <code>errors</code>, <code>isValid</code>, <code>isSubmitting</code>…</div></div>
<div class="deflist__row"><div class="deflist__t">watch</div><div class="deflist__d">Observa el valor de un campo en tiempo real (lógica condicional).</div></div>
<div class="deflist__row"><div class="deflist__t">reset</div><div class="deflist__d">Restablece los valores y el estado del formulario.</div></div>
</div>

## Ejemplo básico

```jsx
import { useForm } from 'react-hook-form';

function MiFormulario() {
  const { register, handleSubmit, formState: { errors } } = useForm();

  const onSubmit = (data) => {
    console.log('Datos del formulario:', data);
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input {...register('name', { required: 'El nombre es requerido' })} />
      {errors.name && <p>{errors.name.message}</p>}

      <input {...register('email', { required: 'El email es requerido' })} />
      {errors.email && <p>{errors.email.message}</p>}

      <button type="submit">Enviar</button>
    </form>
  );
}
```

<Callout type="info" title="¿Qué es ...register?">
La función `register` devuelve un objeto con todo lo necesario para controlar el campo (`name`, `onChange`, `ref`…). El **spread** `{...register('name')}` lo “esparce” como props del `<input>` automáticamente. Por eso **no** necesitás `value` ni `onChange` a mano.
</Callout>

### Qué pasa al enviar

`handleSubmit` primero **valida** todos los campos registrados. Sólo si están OK ejecuta tu `onSubmit`; si no, llena `formState.errors` y muestra los mensajes — sin que el `onSubmit` llegue a correr.

## Reglas de validación

RHF se apoya en los estándares de validación de HTML. Las reglas se pasan como segundo argumento de `register`:

<div class="deflist">
<div class="deflist__row"><div class="deflist__t">required</div><div class="deflist__d">Campo obligatorio.</div></div>
<div class="deflist__row"><div class="deflist__t">minLength / maxLength</div><div class="deflist__d">Longitud mínima / máxima (objeto con <code>value</code> y <code>message</code>).</div></div>
<div class="deflist__row"><div class="deflist__t">min / max</div><div class="deflist__d">Valor numérico mínimo / máximo.</div></div>
<div class="deflist__row"><div class="deflist__t">pattern</div><div class="deflist__d">Una expresión regular que debe cumplir (ej. formato de email).</div></div>
<div class="deflist__row"><div class="deflist__t">validate</div><div class="deflist__d">Función personalizada: devuelve <code>true</code> o un mensaje de error.</div></div>
</div>

```jsx
// required con mensaje
{...register('name', { required: 'El nombre es requerido' })}

// minLength
{...register('password', {
  minLength: { value: 8, message: 'Mínimo 8 caracteres' },
})}

// validate: función propia (ej. confirmar contraseña)
{...register('confirmacion', {
  validate: (value) => value === password || 'Las contraseñas no coinciden',
})}

// pattern con regex (ej. email)
{...register('email', {
  required: 'El email es requerido',
  pattern: {
    value: /^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z]{2,4}$/i,
    message: 'Ingresá un email válido',
  },
})}
```

### Mini-repaso de expresiones regulares

Una **regex** es un patrón de búsqueda. Los **metacaracteres** más comunes:

<div class="deflist">
<div class="deflist__row"><div class="deflist__t">^ … $</div><div class="deflist__d">Inicio y fin de la cadena.</div></div>
<div class="deflist__row"><div class="deflist__t">.</div><div class="deflist__d">Cualquier carácter (menos salto de línea).</div></div>
<div class="deflist__row"><div class="deflist__t">* + ?</div><div class="deflist__d">Cero o más · una o más · cero o una repetición.</div></div>
<div class="deflist__row"><div class="deflist__t">[...]</div><div class="deflist__d">Conjunto de caracteres permitidos. Ej: <code>[A-Za-z0-9]</code>.</div></div>
<div class="deflist__row"><div class="deflist__t">\d</div><div class="deflist__d">Un dígito (equivale a <code>[0-9]</code>).</div></div>
</div>

```text
/\d+/                 → uno o más dígitos: "42", "12345"
/^[A-Za-z0-9]+$/      → solo letras y números, sin espacios ni símbolos
```

## formState: el pulso del formulario

`formState` informa el estado actual del formulario para reaccionar a las interacciones del usuario:

<div class="deflist">
<div class="deflist__row"><div class="deflist__t">isDirty</div><div class="deflist__d">El usuario modificó algún campo respecto del valor inicial.</div></div>
<div class="deflist__row"><div class="deflist__t">isValid</div><div class="deflist__d">Todos los campos cumplen las reglas de validación.</div></div>
<div class="deflist__row"><div class="deflist__t">isSubmitting</div><div class="deflist__d">El formulario se está enviando (ideal para deshabilitar el botón).</div></div>
<div class="deflist__row"><div class="deflist__t">isSubmitted</div><div class="deflist__d">El formulario se envió al menos una vez.</div></div>
<div class="deflist__row"><div class="deflist__t">errors</div><div class="deflist__d">Objeto con los errores de validación por campo.</div></div>
</div>

```jsx
const { register, handleSubmit, formState } = useForm();
const { isValid, isSubmitting, errors } = formState;

<button type="submit" disabled={!isValid || isSubmitting}>
  {isSubmitting ? 'Enviando...' : 'Enviar'}
</button>
```

<Callout type="tip" title="Mejor experiencia de usuario">
Con `formState` podés **deshabilitar** el botón mientras se envía (`isSubmitting`), evitar enviar si el form es inválido (`isValid`) y mostrar errores solo después de que el usuario interactuó. Todo sin escribir lógica de estado a mano.
</Callout>

<Callout type="parcial">
- **React Hook Form** simplifica formularios con hooks (menos código, menos re-renders).
- Todo sale de **`useForm`**: `register`, `handleSubmit`, `formState`, `watch`, `reset`.
- **`register`** conecta el campo y define reglas; se aplica con **spread** `{...register('campo', reglas)}`.
- **`handleSubmit(onSubmit)`** valida y, si está OK, llama a `onSubmit` con los datos.
- Reglas: `required`, `minLength`, `maxLength`, `min`, `max`, **`pattern`** (regex), `validate`.
- **`formState`**: `errors`, `isValid`, `isDirty`, `isSubmitting`, `isSubmitted`.
- Los errores se muestran con `errors.campo.message`.
</Callout>
