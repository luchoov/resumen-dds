- **Padre → Hijo:** por **props** (datos, funciones, lo que sea).
- **Hijo → Padre:** el padre define una **función callback** y se la pasa al hijo; el hijo la **invoca**.

```jsx
// Padre: tiene el estado y define el callback
function ListaMensajes() {
  const [mensajes, setMensajes] = useState([]);
  const agregarMensaje = (texto) => setMensajes([...mensajes, texto]);

  return <NuevoMensaje onEnviar={agregarMensaje} />;
}

// Hijo: invoca el callback recibido
function NuevoMensaje({ onEnviar }) {
  const [texto, setTexto] = useState('');
  return (
    <>
      <input value={texto} onChange={(e) => setTexto(e.target.value)} />
      <button onClick={() => onEnviar(texto)}>Enviar</button>
    </>
  );
}
```

<Callout type="tip" title="Convención onAlgo / props.onAlgo">
Nombrá las funciones como **`onAlgo`** en el padre y usalas como **`props.onAlgo`** en el hijo. Mejora muchísimo la legibilidad y deja claro el flujo de eventos.
</Callout>

### Context como atajo

Cuando el dato es global (usuario, tema, idioma), **Context** evita pasar props por cada nivel. El padre **provee** y cualquier descendiente **consume** con `useContext`:

```jsx
const UsuarioContext = createContext();

function App() {
  const [usuario, setUsuario] = useState(null);
  return (
    <UsuarioContext.Provider value={{ usuario, setUsuario }}>
      <Header />
    </UsuarioContext.Provider>
  );
}

function Header() {
  const { usuario } = useContext(UsuarioContext);
  return <div>Bienvenido: {usuario?.nombre || 'Invitado'}</div>;
}
```

```jsx
function FormularioControlado() {
  const [nombre, setNombre] = useState('');

  const handleSubmit = (e) => {
    e.preventDefault();
    alert(`Nombre: ${nombre}`);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input value={nombre} onChange={(e) => setNombre(e.target.value)} />
      <button type="submit">Enviar</button>
    </form>
  );
}
```

## Componentes no controlados

Un **componente no controlado** deja que el **DOM** maneje el valor. React solo lo **lee cuando lo necesita**, usando una **ref**:

```jsx
import { useRef } from 'react';

function FormularioNoControlado() {
  const inputRef = useRef(null);

  const handleSubmit = (e) => {
    e.preventDefault();
    alert(`Nombre: ${inputRef.current.value}`); // se lee del DOM
  };

  return (
    <form onSubmit={handleSubmit}>
      <input ref={inputRef} />
      <button type="submit">Enviar</button>
    </form>
  );
}
```

### Controlado vs No controlado

| Aspecto | Controlado | No controlado |
|---|---|---|
| Fuente de verdad | Estado de React | El DOM |
| Acceso al valor | `value` + `onChange` | `ref.current.value` |
| Validación | Sencilla y reactiva | Manual |
| Sincronización | Siempre con el estado | Parcial |

<Callout type="analogia" title="¿Quién es el dueño del input?">
**Controlado:** React es el **dueño** — cada tecla pasa por el estado. **No controlado:** el **DOM** guarda el valor y React lo “espía” solo cuando hace falta.
</Callout>

<Callout type="tip" title="Recomendación de React">
Usá **controlados por defecto**: son más predecibles, seguros y fáciles de validar. Dejá los **no controlados** para casos puntuales (formularios de terceros, performance extrema).
</Callout>

## onChange con múltiples campos

Para un formulario con varios campos, un solo `handleChange` que usa `[e.target.name]` mantiene todo en un objeto de estado:

```jsx
function FormularioUsuario() {
  const [form, setForm] = useState({ nombre: '', email: '' });

  const handleChange = (e) => {
    setForm({ ...form, [e.target.name]: e.target.value });
  };

  return (
    <form>
      <input name="nombre" value={form.nombre} onChange={handleChange} />
      <input name="email" value={form.email} onChange={handleChange} />
    </form>
  );
}
```

<Callout type="parcial">
- Padre→hijo por **props**; hijo→padre por **callback** (convención `onAlgo`).
- **Context** comparte datos globales sin pasar props nivel por nivel.
- Formularios: cuidar **accesibilidad**, **validación** y **serialización**.
- **Controlado**: el `value` está atado al **estado** (`value` + `onChange`) → única fuente de verdad.
- **No controlado**: el valor vive en el **DOM**, se lee con **`ref`**.
- React recomienda **controlados** por defecto.
- Varios campos: un `handleChange` con **`[e.target.name]`**.
</Callout>
