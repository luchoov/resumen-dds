## express.Router(): rutas modulares

Meter todas las rutas en `index.js` se vuelve un caos. La clase **`express.Router()`** permite crear manejadores de rutas **modulares y montables**: una especie de **“mini-aplicación”** con su propio sistema de rutas y middlewares, que después se **monta** en la app principal.

```js
// articulos.js  →  el módulo router
const router = express.Router();

router.get('/api/articulos', async (req, res, next) => { /* … */ });
router.get('/api/articulos/:id', async (req, res, next) => { /* … */ });

module.exports = router;
```

```js
// index.js  →  se monta el router en la app
const articulosRouter = require('./routes/articulos');
app.use(articulosRouter);
```

<Callout type="tip" title="¿Por qué modularizar?">
Cada recurso (artículos, usuarios, ventas) tiene su propio archivo router. El `index.js` queda limpio y el proyecto **escala** sin convertirse en un monstruo de 1000 líneas.
</Callout>

## Sirviendo archivos estáticos

Los **archivos estáticos** son recursos que se entregan **tal cual**, sin procesamiento del servidor: imágenes, CSS, JS, fuentes. Se sirven con el middleware `express.static()`:

```js
app.use(express.static('public'));
```

Con esto, una imagen en `public/images/logo.png` queda accesible en:

```text
http://localhost:3000/images/logo.png
```

Notá que la carpeta `public` **no** aparece en la URL: es la raíz desde donde Express busca.

### path y __dirname

Para que la ruta a `public` sea **correcta sin importar desde dónde ejecutes** el script, se combina el módulo `path` con `__dirname`:

<div class="deflist">
<div class="deflist__row"><div class="deflist__t">__dirname</div><div class="deflist__d">Variable global de Node con la ruta <strong>del directorio del archivo actual</strong>.</div></div>
<div class="deflist__row"><div class="deflist__t">path.join()</div><div class="deflist__d">Une segmentos de ruta de forma segura y multiplataforma (Windows / Linux).</div></div>
</div>

```js
const path = require('path');
app.use(express.static(path.join(__dirname, 'public')));
```

<Callout type="info" title="¿Por qué no usar 'public' a secas?">
`'public'` es relativo al **directorio de trabajo** (desde dónde corrés `node`), que puede variar. `path.join(__dirname, 'public')` es relativo al **archivo**, así que siempre apunta al lugar correcto.
</Callout>

## CORS: Cross-Origin Resource Sharing

**CORS** es una **política del navegador** que, por seguridad, **bloquea** que una página de un origen (dominio) acceda a recursos de **otro** origen mediante peticiones AJAX (`fetch`, `XMLHttpRequest`, Axios…).

Para que el backend **autorice** a otros dominios, se instala y configura el middleware `cors`:

```bash
npm i cors
```

```js
const cors = require('cors');

// Opción A: permitir TODOS los orígenes (cómodo, pero abierto)
app.use(cors());

// Opción B: lista blanca — sólo el frontend autorizado
app.use(cors({
  origin: 'http://localhost:5173',
  credentials: true, // necesario si se usan cookies
}));
```

<Callout type="danger" title="cors() abierto ≠ producción">
`app.use(cors())` habilita **cualquier** origen. Está bien para practicar, pero en producción se restringe con una **lista blanca** (`origin`) a los dominios que realmente deben acceder.
</Callout>

<Callout type="analogia" title="CORS es el portero del edificio">
El navegador es un **portero estricto**: por defecto no deja que gente de “otro edificio” (otro dominio) retire paquetes. La cabecera CORS que manda el backend es la **lista de invitados autorizados**. Si tu dominio no está en la lista, el portero te frena.
</Callout>

<Callout type="parcial">
- **`express.Router()`** = mini-app de rutas modulares; se monta con `app.use(router)`.
- **`express.static('public')`** sirve archivos tal cual (la carpeta no aparece en la URL).
- **`__dirname`** + **`path.join()`** = rutas robustas sin importar desde dónde corras el script.
- **CORS** = política del **navegador** que bloquea peticiones entre dominios distintos.
- Se habilita en el **backend** con el middleware `cors` (abierto o con lista blanca).
</Callout>
